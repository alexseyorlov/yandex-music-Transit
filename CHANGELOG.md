# История версий (Changelog) — yandex-music-Transit

Все значимые изменения в проекте фиксируются в этом файле.

## [5.0] — 2026-04-23
### Изменено
- Полностью переработана архитектура экспорта: переход с DOM/скроллинга на API-only подход.
- Добавлен перехват `fetch` для захвата playlist JSON-ответов прямо в активной браузерной сессии.
- Добавлены активные API-пробы для разных форматов URL (`/users/.../playlists/...` и `/playlists/...`).

### Добавлено
- Новый формат статусов панели: `INIT`, `PROBE`, `CAPTURE`, `PARTIAL`, `COMPLETE`, `EMPTY`, `STOPPED`.
- Кнопки управления `Save TXT` и `Stop`.
- Обновленный экспортный заголовок с версией, названием плейлиста, ID и счетчиками `Captured/Expected`.
- Обновленный `README.md` с документацией для версии `v5.0 API-Only`, ссылкой на альтернативный сторонний проект и атрибуцией.

### Удалено
- Двухфазный скролл-алгоритм `Gravity Drift` и зависимость от виртуализированного DOM.

## [4.4] — 2026-04-15
### Добавлено
- Внедрен алгоритм **"Gravity Drift"**: двухфазное сканирование (быстрый спуск и точный подъем).
- Реализована система "докачки" данных: подъем вверх со скоростью 700px для фиксации пропущенных DOM-узлов.
- Ускорена фаза спуска до 3600px за итерацию (x2 от предыдущей версии).

### Исправлено
- Проблема потери треков при быстрой прокрутке из-за особенностей рендеринга библиотеки Virtuoso.
- Улучшена стабильность UI-панели на экранах с разным разрешением.

## [4.0] — 2026-04-14
### Добавлено
- Полностью новый интерфейс с использованием темы **Acid Minimal**.
- Добавлен прогресс-бар и счетчик найденных треков в реальном времени.
- Механизм автоматической остановки при достижении конца плейлиста (Stuck-detection).

## [3.1] — 2026-04-10
### Добавлено
- Улучшена логика сбора данных: теперь скрипт ищет только прямые ссылки на треки и артистов, исключая рекламный мусор.
- Сортировка итогового списка по алфавиту.
- Автоматическое удаление дубликатов с помощью объекта `Map`.

## [2.1] — 2026-04-05
### Добавлено
- Базовый алгоритм скроллинга плейлиста.
- Генерация и скачивание `.txt` файла прямо из браузера.

## [1.0] — 2026-04-01
### Добавлено
- Первая рабочая концепция: ручной сбор данных из открытого DOM-дерева плейлиста "Мне нравится".

---
*Проект развивается в рамках поиска максимально быстрого способа миграции медиатеки Яндекс.Музыка
